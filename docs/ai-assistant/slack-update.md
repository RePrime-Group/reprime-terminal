*Deal AI Assistant — Update*

Hey team 👋 quick update on where the Deal AI Assistant landed this week. Phase 4 is functionally complete and we're effectively production-ready end-to-end. Posting this so everyone's on the same page.

*What's live*

- *Real backend wired up.* Chat, conversation history, and feedback are all hitting the n8n + Supabase stack instead of mocks.
- *Three n8n workflows are active:* `deal-assistant-chat` (the main one), `deal-assistant-feedback`, and `deal-assistant-custom-tools`. The history workflow was deactivated because direct Supabase reads from the API route are simpler and faster.
- *Conversation titles* are auto-generated by a Claude Haiku call in the background after the first turn, so the sidebar shows clean 4 to 6 word titles instead of truncated user messages.
- *Tools the agent can call:* Get Deal, Get Tenants, Get Documents, and Get Addresses. Each is gated to the locked deal so the assistant stays scoped.

*Portfolio deals (the big fix)*

- *Earlier the assistant was returning wrong tenants* for portfolio deals (e.g. asking about Three Notch Plaza was returning Frayser Village tenants).
- Root cause was a combination of (a) the Get Addresses tool being disconnected from the agent, (b) noise in the response that confused the model, and (c) the prompt not teaching it how to map building names to address IDs.
- All three are fixed now. The assistant correctly scopes to the right building, says "only 2 tenants exist" when that's the truth, and lists actual building labels when the user names a building that doesn't exist.

*System prompt*

- Split into a static prefix and a dynamic suffix so we can enable Anthropic prompt caching later without re-architecting.
- Source of truth is now in two places (kept in sync): `src/lib/ai/prompts/system.static.ts` and `n8n/deal-assistant-prompt.md`.
- Refresh covers identity, tool routing, portfolio handling, refusal patterns, and an explicit guard against em-dashes and prompt injection.

*UI polish*

- Replaced the three pulsing dots with a gold sparkle SVG that pulses and shimmers, plus rotating status text in plain English ("Thinking", "Crunching the numbers", etc.) instead of raw i18n keys leaking through.
- Markdown rendering is properly sanitized and styled for the dark theme.
- Stream/replay bug fixed so only the freshly-arrived assistant message animates.

*Security and reliability*

- *Patched a silent-failure path:* non-existent deal IDs were returning empty 200 instead of 403. The Check Entitlement node now always emits an item so the IF gate routes correctly.
- Workflow timeout capped at 60 seconds, agent capped at 5 tool-use iterations.
- Audit table records every chat turn (user, deal, conversation, model, payload).

*QA done*

- 27 of 28 automated checks passing across preflight, tool routing, portfolio scoping, multi-turn memory, identity/style guards, cross-deal isolation, prompt injection, and error paths. The one "failure" was a test-side assertion bug, not an AI bug, the model correctly identified an even-earlier expiry that the assertion didn't anticipate.
- Test data was hard-deleted after the run so nothing is left polluting the conversation history.

*What's documented*

- `docs/ai-assistant/runbook.md` covers deploy steps, kill switch, key rotation, workflow IDs, and known caveats.
- `docs/ai-assistant/api-contract.md` is the source of truth for the API shape.
- All four workflow JSONs are checked into `n8n/workflows/` with credentials stripped, so reviewers can see the n8n state in a PR.

*What's still on the radar*

- *Prompt caching at runtime* is structured-ready (verified hitting at the API level) but n8n's langchain Anthropic node doesn't yet pass `cache_control`, so production isn't capturing the savings yet. Logged in the runbook as a follow-up.
- Token-count metrics are still chars/4 estimates; switching to real Anthropic counts is a small follow-up.
- Voice support (Phase 6) is scaffolded but not active.

Happy to walk anyone through the architecture or run a live demo. Thanks for the patience while we sorted the portfolio bug 🙏
